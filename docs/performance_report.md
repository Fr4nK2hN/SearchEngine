# 离线效果与在线性能评估

## 算法与策略

- Baseline：使用 ES 原分数排序。
- LTR（LambdaMART）：特征标准化与训练，支持特征重要性分析（`ranking/ranker.py:221-251`）。
- Cross‑Encoder：对查询‑文本对进行强相关性评分（教师/对照）。
- Hybrid：先用 LTR 排序，再对 Top‑10 用 CE 微调，融合权重 0.6/0.4。

### 指标说明

- NDCG@K
  - 归一化折扣累积增益，针对分级相关性（如 0/1/2/3），越靠前的高相关文档贡献越大，范围 0–1。
  - 适合强调“前列质量”的排序评估。
- MRR
  - 第一个相关文档的倒数排名的平均值，例如第 1 位命中为 1.0，第 5 位命中为 0.2；未命中为 0。
  - 适用于“只要一个好答案”的任务（问答、导航查询）。
- Precision@K
  - 前 K 个结果中“相关”的比例，衡量列表纯度，不考虑位置（只看是否相关）。
- Recall@K
  - 前 K 个结果覆盖“所有相关文档”的比例，衡量覆盖率；依赖相关文档总量。
- MAP
  - 所有查询的平均 AP（AP 为每个查询“相关位置上的 Precision 的平均”），强调将所有相关文档尽量排到靠前。
  - 适合二值相关性、希望整体相关都尽量靠前的场景。

## 在线性能评估（最近窗口）

- 要点：
  - LTR 在均值上优于 CE；Hybrid 在 Top‑10 下实现更低均值。
  - P99 尾部主要来自偶发检索慢或未命中文档向量导致在线补算；随向量覆盖率提升，P99 将继续下探。

### 在线指标说明

- 平均检索 (ms)
  - `retrieval_ms` 的均值，表示 Elasticsearch 检索阶段耗时。
- 特征 (ms)
  - `feature_ms` 的均值，表示 LTR 特征提取阶段耗时；Cross‑Encoder 与 Baseline 设为 0。
- 推理 (ms)
  - `inference_ms` 的均值，表示模型前向推理耗时；LTR/CE/Hybrid 均记录，Baseline 为 0。
- 平均总耗时 (ms)
  - `total_ms` 的均值，端到端总耗时（检索 + 特征 + 推理 + 其他开销）。
- P50 / P90 / P95 / P99 (ms)
  - 基于 `total_ms` 的分位数统计，分别代表中位数、90%/95%/99% 尾部表现。
- 统计窗口与分组
  - 窗口：`--last N` 仅统计最后 N 行日志，入口：`tools/analyze_latency.py:35-49`。
